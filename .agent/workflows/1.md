**Name:** AI News Control Center
**Goal:** A centralized dashboard to manage automated content publishing workflows (n8n pipelines) for multiple platforms (Telegram, VK, OK, Website).
**Key Functionality:**
- Managing publishing "Recipes" (timings, active status).
- Editing prompts for AI agents.
- Monitoring content status.

# ðŸ— ARCHITECTURE & STRUCTURE
- **Root:** `ainews-control-center.fresh`
- **Framework:** Next.js 14+ (App Router).
- **Routing:**
  - `/recipes` -> Publishing configuration (Platform active/delay).
  - `/prompts` -> AI System Prompts management.
  - `/content` -> News feed & status monitoring.
- **Key Directories:**
  - `app/(dashboard)/...` -> Main app pages protected by auth.
  - `components/ui` -> Shadcn UI primitives.
  - `lib/supabase` -> Database clients.

# ðŸ—„ DATABASE SCHEMA (Supabase)
**Table: `publish_recipes`** (Source of Truth for Automation)
- `id` (uuid)
- `platform` (text): 'tg', 'vk', 'ok', 'site'
- `is_active` (bool): Master switch for the channel.
- `is_main` (bool): Flag for the primary source channel.
- `delay_hours` (int): Publishing delay time.

# ðŸŽ¨ UI/UX STANDARDS
**Platform Identity (Strict Colors):**
- **Telegram (TG):** Blue (`text-blue-400`, `bg-blue-50`)
- **VKontakte (VK):** Deep Blue (`text-blue-600`)
- **Odnoklassniki (OK):** Orange (`text-orange-500`)
- **Website (Site):** Emerald/Green (`text-emerald-500`)

**Components Behavior:**
- **Inputs:** Must always have `bg-white text-black` to prevent dark mode conflicts.
- **Save Actions:** Use "Dirty State" logic (Show Apply/Cancel only on change).

# ðŸ”„ N8N INTEGRATION
- The frontend **DOES NOT** publish content directly.
- The frontend **UPDATES** the Supabase DB.
- **n8n workflows** poll the Supabase DB or listen to webhooks to trigger actual posting based on `publish_recipes` settings.
